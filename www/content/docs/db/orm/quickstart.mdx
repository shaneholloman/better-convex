Get up and running with Better‑Convex ORM in a few minutes. This assumes you already have a Convex project.

## Prerequisites

- Node.js 18+
- A Convex project (or [create one](https://docs.convex.dev/quickstart))
- Basic TypeScript familiarity

## Step 2: Define Tables

```ts title="convex/schema.ts"
import { convexTable, text, boolean, id } from 'better-convex/orm';

export const users = convexTable('users', {
  name: text().notNull(),
  email: text().notNull(),
});

export const posts = convexTable('posts', {
  title: text().notNull(),
  content: text().notNull(),
  published: boolean(),
  userId: id('users'),
});
```

<Callout icon={<InfoIcon />}>
**Nullable by default**: `text()` is `string | null`. Use `.notNull()` for required fields.
</Callout>

## Step 3: Define Relations + Edges

```ts title="convex/schema.ts"
import {
  defineRelations,
  extractRelationsConfig,
} from 'better-convex/orm';

export const ormSchema = defineRelations({ users, posts }, (r) => ({
  users: { posts: r.many.posts() },
  posts: {
    author: r.one.users({ from: r.posts.userId, to: r.users._id }),
  },
}));

export const ormEdges = extractRelationsConfig(ormSchema);
```

## Step 4: Query Data

```ts title="convex/queries.ts"
import { query } from './_generated/server';
import { createDatabase } from 'better-convex/orm';
import { ormSchema, ormEdges } from './schema';

export const getPublishedPosts = query({
  handler: async (ctx) => {
    const db = createDatabase(ctx.db, ormSchema, ormEdges);

    return db.query.posts.findMany({
      where: { published: true },
      orderBy: { _creationTime: 'desc' },
      limit: 10,
      with: { author: true },
    });
  },
});
```

## What You Learned

- Defined tables with `convexTable()`
- Added relations with `defineRelations()`
- Created a typed DB instance with `createDatabase()`
- Queried data using Drizzle‑style `findMany()`

## Common Gotchas

| Issue | Fix |
|---|---|
| `createDatabase` type errors | Pass `ormSchema` + `ormEdges`, not raw tables |
| Relations not loading | Ensure `defineRelations()` and `extractRelationsConfig()` are used |
| Slow `orderBy` | Add `.index()` for the first order field |
| `columns` doesn’t reduce reads | Projection is post‑fetch |
