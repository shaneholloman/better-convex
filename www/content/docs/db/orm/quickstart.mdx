Get up and running with Better-Convex ORM in 5 minutes. This guide assumes you have an existing Convex project.

## Prerequisites

- Node.js 18+ installed
- Existing Convex project (or [create one](https://docs.convex.dev/quickstart))
- Basic familiarity with TypeScript

## Step 2: Define Your Schema

Create tables using the Drizzle-style `convexTable()` function:

```ts title="convex/schema.ts" showLineNumbers {1-2,4-11}
import { convexTable, relations, text, boolean } from 'better-convex/orm';

// Define tables
const users = convexTable('users', {
  name: text().notNull(),
  email: text().notNull(),
});

const posts = convexTable('posts', {
  title: text().notNull(),
  content: text().notNull(),
  published: boolean(),
});

export const schema = {
  users,
  posts,
};

export default schema;
```

<Callout icon={<InfoIcon />}>
**Column Builders**: Better-Convex ORM uses Drizzle-style column builders (`text()`, `integer()`, `boolean()`) for schema definition. This provides familiar Drizzle ergonomics with Convex's type inference.
</Callout>

## Step 3: Define Relations

Use Drizzle's `relations()` API to connect tables:

```ts title="convex/schema.ts" showLineNumbers {18-29}
import { convexTable, relations, text, boolean } from 'better-convex/orm';

const users = convexTable('users', {
  name: text().notNull(),
  email: text().notNull(),
});

const posts = convexTable('posts', {
  title: text().notNull(),
  content: text().notNull(),
  published: boolean(),
});

// Define 1:many relationship (user has many posts)
const usersRelations = relations(users, ({ many }) => ({
  posts: many(posts),
}));

// Define many:1 relationship (post belongs to user)
const postsRelations = relations(posts, ({ one }) => ({
  author: one(users, {
    fields: [posts.userId],
    references: [users._id],
  }),
}));

export const schema = {
  users,
  posts,
  usersRelations,
  postsRelations,
};

export default schema;
```

<Callout icon={<CheckCircle />}>
**✅ 100% Drizzle Compatible**: The relations API is identical to Drizzle ORM. Code that works in Drizzle works in Better-Convex.
</Callout>

## Step 4: Query Your Data

Create Convex queries using the ORM:

```ts title="convex/queries.ts" showLineNumbers {1-3,7-11}
import { query } from './_generated/server';
import { db } from 'better-convex/orm';
import { eq } from 'better-convex/orm/filter';

export const getPublishedPosts = query({
  handler: async (ctx) => {
    const posts = await db(ctx).query.posts.findMany({
      where: eq(posts.published, true),
      limit: 10,
    });

    return posts;
  },
});

export const getAllUsers = query({
  handler: async (ctx) => {
    const users = await db(ctx).query.users.findMany();
    return users;
  },
});
```

**Generated Convex Query:**
```ts
// Behind the scenes, Better-Convex compiles to:
ctx.db.query("posts")
  .filter(q => q.eq(q.field("published"), true))
  .take(10)
```

**Type Inference:**
```ts
// Return type is automatically inferred from your schema:
Array<{
  _id: Id<'posts'>,
  title: string,
  content: string,
  published: boolean,
  userId: Id<'users'>,
  _creationTime: number
}>
```

## Common Gotchas

| Issue | Solution |
|-------|----------|
| **Type errors on `db(ctx)`** | Make sure you imported `db` from `better-convex/orm`, not `convex` |
| **Relations not working** | Export both the table AND the relations object in your schema |
| **Missing fields in queries** | Convex always returns full documents - you can't select specific columns like in SQL |
| **Undefined in TypeScript** | Convex uses `undefined` for missing fields, not `null` like SQL databases |
