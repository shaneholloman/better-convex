Get up and running with Better-Convex ORM in 5 minutes. This guide assumes you have an existing Convex project.

## Prerequisites

- Node.js 18+ installed
- Existing Convex project (or [create one](https://docs.convex.dev/quickstart))
- Basic familiarity with TypeScript

## Step 2: Define Your Tables

Create tables using the Drizzle-style `convexTable()` function:

```ts title="convex/schema.ts" showLineNumbers {1-2,4-13}
import { convexTable, text, boolean, id } from 'better-convex/orm';

const users = convexTable('users', {
  name: text().notNull(),
  email: text().notNull(),
});

const posts = convexTable('posts', {
  title: text().notNull(),
  content: text().notNull(),
  published: boolean(),
  userId: id('users'),
});
```

<Callout icon={<InfoIcon />}>
**Column Builders**: Better-Convex ORM uses Drizzle-style column builders (`text()`, `integer()`, `boolean()`, `id()`) for schema definition. This provides familiar Drizzle ergonomics with Convex's type inference.
</Callout>

## Step 3: Define Relations and Build Schema

Use Drizzle's `relations()` API to connect tables, then build a typed schema for the query builder:

```ts title="convex/schema.ts" showLineNumbers {1-3,5-23,25-33}
import {
  convexTable,
  relations,
  text,
  boolean,
  id,
  buildSchema,
  extractRelationsConfig,
} from 'better-convex/orm';

const users = convexTable('users', {
  name: text().notNull(),
  email: text().notNull(),
});

const posts = convexTable('posts', {
  title: text().notNull(),
  content: text().notNull(),
  published: boolean(),
  userId: id('users'),
});

const usersRelations = relations(users, ({ many }) => ({
  posts: many(posts),
}));

const postsRelations = relations(posts, ({ one }) => ({
  author: one(users, {
    fields: [posts.userId],
    references: [users._id],
  }),
}));

export const schema = {
  users,
  posts,
  usersRelations,
  postsRelations,
};

export const ormSchema = buildSchema(schema);
export const ormEdges = extractRelationsConfig(ormSchema);
```

<Callout icon={<CheckCircle />}>
**✅ Drizzle-Compatible**: The relations API is identical to Drizzle ORM. Code that works in Drizzle works in Better-Convex (use `_id` instead of `id`).
</Callout>

## Step 4: Query Your Data

Create Convex queries using the ORM:

```ts title="convex/queries.ts" showLineNumbers {1-3,7-14}
import { query } from './_generated/server';
import { createDatabase } from 'better-convex/orm';
import { ormSchema, ormEdges } from './schema';

export const getPublishedPosts = query({
  handler: async (ctx) => {
    const db = createDatabase(ctx.db, ormSchema, ormEdges);

    const posts = await db.query.posts.findMany({
      where: (posts, { eq }) => eq(posts.published, true),
      limit: 10,
    });

    return posts;
  },
});
```

**Generated Convex Query:**
```ts
// Behind the scenes, Better-Convex compiles to:
ctx.db.query('posts')
  .filter(q => q.eq(q.field('published'), true))
  .take(10)
```

**Type Inference:**
```ts
// Return type is automatically inferred from your schema:
Array<{
  _id: Id<'posts'>,
  title: string,
  content: string,
  published: boolean | null,
  userId: Id<'users'> | null,
  _creationTime: number
}>
```

## Common Gotchas

| Issue | Solution |
|-------|----------|
| **`createDatabase` type errors** | Pass `buildSchema(schema)` and `extractRelationsConfig()` output, not raw tables |
| **Relations not loading** | Export relations and pass `ormEdges` into `createDatabase` |
| **Slow `orderBy`** | Add `.index()` for fields you sort by frequently |
| **`columns` doesn't reduce reads** | Column selection is post-fetch projection in Convex |
| **Nullable fields in filters** | Use `isNull()` / `isNotNull()` instead of comparing to `null` |
